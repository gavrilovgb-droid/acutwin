## 1. Вход и выход

**Войти:**
1. Откройте программу в браузере.
2. Введите ваш логин и пароль.
3. Нажмите **«Войти»**.

**Выйти:** нажмите кнопку **«Выйти»** в правом верхнем углу.

> Если закрыли вкладку — при следующем открытии программа попросит войти заново.

---

## 2. Новый приём

Главная страница — форма регистрации нового пациента.

**Заполните поля:**

| Поле | Что вводить |
|------|-------------|
| **Дата и время** | Заполняется автоматически, можно изменить |
| **ФИО** | Начните вводить фамилию — программа подскажет из базы |
| **Возраст** | Полных лет |
| **Пол** | Мужской / Женский |
| **Причина обращения** | Основная жалоба |

Под полем причины обращения есть **быстрые кнопки** — частые жалобы. Нажмите нужные, они добавятся к тексту автоматически.

Когда всё заполнено — нажмите **«К тактике лечения»**.

---

## 3. Параметры сеанса ИРТ

В верхней части страницы тактики лечения находится полоска с параметрами сеанса. Заполняйте её **в процессе приёма** — до и после воздействия.

```
Боль до [0–10]   Боль после [0–10]   Воздействие [▾]   Стимуляция [▾]   Экспозиция, мин [20]   □ Де-ци
```

| Поле | Что вводить |
|------|-------------|
| **Боль до** | Интенсивность боли по шкале НРС перед сеансом (0 — нет боли, 10 — невыносимая) |
| **Боль после** | Интенсивность боли сразу после сеанса |
| **Воздействие** | Вид воздействия: корпоральная, аурикулярная, Су-Джок и др. |
| **Стимуляция** | Тонизирование / Седатирование / Гармонизация |
| **Экспозиция, мин** | Время нахождения игл (по умолчанию 20 мин) |
| **Де-ци** | Отметьте галочкой, если пациент почувствовал ощущение де-ци (распирание, тепло, прохождение тока) |

> **Зачем это нужно:** данные НРС накапливаются в карточке пациента и строят график динамики боли по курсу. Это позволяет объективно показать пациенту прогресс лечения.

## 4. Тактика лечения

Пять инструментов для подбора точек. Переключайтесь между ними с помощью вкладок вверху.

---

### 🔹 Выбор точек по показаниям

Подбор точек по симптому или диагнозу.

1. Введите симптом в поиск или выберите из списка.
2. Программа покажет точки с доказательной базой — рядом с диагнозом будет значок **Grade A / B / C** (см. раздел 5).
3. Отметьте нужные точки галочками.
4. Нажмите **«Добавить в рецепт»**.

---

### 🔹 Рецепты ТКМ

Готовые классические рецепты традиционной китайской медицины.

1. Выберите категорию (синдром, заболевание).
2. Нажмите на нужный рецепт — откроется список точек с действием (тон/сед).
3. Нажмите **«Добавить в рецепт»**, чтобы применить к этому приёму.
4. Нажмите **🖨️** рядом с рецептом, чтобы сразу распечатать.

---

### 🔹 Подбор по У-Син

Подбор точек по теории пяти элементов.

**По симптомам:**
1. Отметьте симптомы, которые есть у пациента.
2. Программа определит доминирующий элемент и покажет рекомендованные точки.
3. Нажмите **«Распечатать»**, чтобы сохранить лист.

**По заболеванию:**
1. Введите заболевание в поиск или выберите из списка.
2. Программа покажет рекомендованные рецепты с точками.
3. Нажмите на рецепт → **🖨️** — распечатать.

---

### 🔹 Меридианная диагностика

Анализ состояния меридианов по жалобам пациента.

1. Отметьте симптомы по меридианам.
2. Программа рассчитает карту состояния меридианов.
3. Нажмите **🖨️**, чтобы распечатать диагностику.
4. Нажмите **«В карточку»**, чтобы сохранить результат.

---

### 🔹 Свой рецепт

Составляете набор точек вручную.

1. Введите название рецепта.
2. Вводите коды точек в поле поиска.
3. Для каждой точки укажите: тонизировать или седировать.
4. Нажмите **«Сохранить рецепт»**.

---

## 5. Доказательная база и МКБ-10

Рядом с каждым заболеванием в списке вы увидите:

- **Код МКБ-10** — международный код диагноза (например, `M54.5` — боль в пояснице). Используйте его при оформлении документации.
- **Grade A / B / C** — уровень доказательности применения иглорефлексотерапии при данном диагнозе:

| Значок | Что означает |
|--------|-------------|
| 🟡 **Grade A** | Высокая доказательность — систематические обзоры и мета-анализы подтверждают эффективность |
| 🟡 **Grade B** | Умеренная доказательность — отдельные качественные исследования |
| **Grade C** | Ограниченная доказательность — клинические наблюдения |
| *(нет значка)* | Данных недостаточно или они отсутствуют |

> Значки помогают объяснить пациенту научную обоснованность лечения и выбрать диагнозы с наилучшей доказательной базой.

## 6. Добавить рецепт в карточку

Из любого инструмента тактики лечения сохраните рецепт в карточку пациента.

- Кнопка называется **«В карточку»** или **«Сохранить рецепт»**.
- После нажатия рецепт появится в разделе «История» карточки этого пациента.

---

## 7. Распечатать лист приёма

В каждом инструменте есть кнопка печати — **🖨️** или «Распечатать».

1. Нажмите кнопку печати.
2. Откроется стандартное окно браузера для печати.
3. Выберите принтер или «Сохранить как PDF».
4. Нажмите «Печать».

> Совет: «Сохранить как PDF» → отправить пациенту в мессенджер.

---

## 8. Карточка пациента

Открывается при нажатии на имя пациента на главной странице.

В карточке:
- **Общая информация** — ФИО, возраст, пол
- **График боли** — динамика НРС по всем сеансам курса (строится автоматически из параметров сеанса)
- **Статистика** — количество приёмов, даты первого и последнего визита, снижение боли в %
- **Рецепты** — все сохранённые наборы точек
- **Кнопка «Назначить рецепт»** — быстро открыть тактику лечения для этого пациента
- **Кнопка «Экспорт в PDF»** — сохранить карточку целиком

---

## 9. История приёмов

Нажмите **«История»** в меню.

**Как найти нужный приём:**
- **Поиск** — введите ФИО пациента
- **Фильтр по врачу** — выберите из списка
- **Фильтр по дате** — укажите период «с» и «по»

Нажмите на строку приёма, чтобы открыть детали.

---

## Если что-то пошло не так

| Проблема | Что сделать |
|----------|-------------|
| Не получается войти | Проверьте Caps Lock; обратитесь к администратору |
| Страница зависла | Обновите страницу (F5) — данные сохраняются автоматически |
| Не печатает | Проверьте, что всплывающие окна не заблокированы в браузере |
| Пациент исчез из списка | Используйте поиск по ФИО на главной странице |

> По всем вопросам обращайтесь к администратору системы.
